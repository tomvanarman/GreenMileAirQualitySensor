# User Story guidelines

## 1. We will be using the following template for our user stories:

<details>
<summary> Click here to view the template </summary>
As a user, I want to ..., so that ... (delete this line, place the text in the title)

[description of the user story and context]

**Tasks**

This user story is divided into a number of tasks. Complete the list below yourself.

- [ ] Task 1.

- [ ] Task 2.

- [ ] ...

**Acceptance criteria**

Acceptance criteria are specific requirements that the User Story must meet. These are usually
unique to each User Story.

- [ ] Acceptance criterion 1

- [ ] Acceptance criterion 2

- [ ] ...

**Definition of Done**

- [ ] All acceptance criteria for the user story have been checked off.

- [ ] Your work has been documented.

- [ ] You have tested your work.

- [ ] The system architecture has been updated.

- [ ] The code has been written with Google code conventions

</details>
